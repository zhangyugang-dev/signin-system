# 会务签到系统 MVP 需求说明

## 1. 项目目标
开发一个最小可用的会务签到系统，用于单场活动的现场签到管理，支持管理员创建并管理一场活动，参会人通过签到页完成签到，管理员可查看签到结果并导出 CSV。

## 2. 目标用户
### 2.1 管理员
- 创建和配置活动
- 导入参会名单
- 查看签到结果
- 导出签到数据

### 2.2 参会人
- 访问签到页面
- 输入姓名和手机号后四位完成签到
- 查看签到成功提示

## 3. 范围定义

## 3.1 本期范围（MVP）
- 单活动版
- 管理员可创建一场活动（活动名称、活动时间）
- 管理员可导入参会名单（CSV）
- 系统生成签到链接
- 参会人通过签到页输入“姓名 + 手机号后4位”签到
- 仅名单内用户允许签到
- 已签到用户不可重复签到
- 管理员可查看签到列表和统计数据
- 管理员可导出签到结果为 CSV

## 3.2 暂不包含
- 多活动管理
- 微信登录 / 短信验证码
- 座位管理
- 签退功能
- 人脸识别
- 大屏展示
- 复杂权限系统

## 4. 核心业务规则
1. 系统初版仅支持单场活动。
2. 管理员需先导入参会名单，名单字段至少包含：姓名、手机号。
3. 参会人签到时输入姓名和手机号后四位。
4. 系统根据“姓名 + 手机号后四位”匹配名单。
5. 若匹配成功且未签到，则签到成功并记录签到时间。
6. 若已签到，则提示不可重复签到。
7. 若匹配失败，则提示信息不正确或不在名单内。

## 5. 页面与功能

### 5.1 管理后台
#### 页面 1：活动配置页
- 输入活动名称
- 输入活动时间
- 显示签到链接
- 上传名单 CSV

#### 页面 2：签到管理页
- 显示总人数
- 显示已签到人数
- 显示未签到人数
- 展示签到名单列表
- 支持导出 CSV

### 5.2 参会签到页
- 展示活动名称和时间
- 输入姓名
- 输入手机号后四位
- 点击签到
- 返回签到结果提示

## 6. 数据结构建议
### 活动 Activity
- id
- name
- eventTime
- createdAt

### 参会人 Attendee
- id
- name
- phone
- phoneLast4
- checkedIn
- checkedInAt

## 7. 验收标准
1. 管理员可成功创建一场活动。
2. 管理员可导入一份 CSV 名单。
3. 参会人可通过姓名 + 手机号后四位完成签到。
4. 非名单成员无法签到。
5. 同一人不能重复签到。
6. 管理后台可查看签到统计和名单。
7. 管理后台可导出签到结果 CSV。
8. 系统可在本地运行并完成部署。

## 8. 技术建议
- 前端：Next.js
- 后端：Next.js API Route
- 数据存储：SQLite（Prisma）
- 部署：Vercel
